Rule-based fuzzy systems: decision-making under uncertainty and the interpretability trade-off

Sarah Etter

12 Oct 2022 • 4 min read

Dr. Jerry Mendel’s seminar on Explainable AI (XAI) for Rule-Based Fuzzy Systems brings to light one of the limitations of fuzzy systems: human interpretability. Mendel describes this problem as being similar to asking a human to perform a p-fold correlation among p variables when most people can only visualize a correlation between two variables. Further, since fuzzy systems operate using natural language-based rules, they are difficult to represent mathematically.

Let's unpack this problem further. For that, we need some context on how fuzzy systems operate.

To illustrate the difficulty in mathematical analysis or interpretation of fuzzy systems, consider a canonical fuzzy system, which is composed of three stages: fuzzification, rule application, and defuzzification [1][2]. In fuzzification, inputs are converted into values of linguistic variables. Unlike traditional crisp logic in set theory where a characteristic function maps inputs to a two-valued boolean corresponding to set membership or non-membership, fuzzy sets use membership functions to map inputs to degrees of membership in the range [0, 1]. Thus in fuzzy systems, an input can belong to multiple fuzzy sets over the range or universe of a linguistic variable. To give a simple example, if we say that an obstacle is near if it is within 1 meter, then an obstacle at 1.1 meters may have a membership value of 0.8 for the fuzzy set representing near, but a membership value of 0.2 for the fuzzy set representing far, since the possibility of being near is much greater than far. In rule application, we infer consequent linguistic variables from the antecedent linguistic variables generated by fuzzification. An example rule is IF an obstacle is near, THEN go slow. If an input belongs in two antecedent fuzzy sets, the membership value of the consequent linguistic variable is computed using the minimum of the antecedent membership values. In defuzzification, we generate crisp outputs from the system by computing the center of gravity of consequent membership values (COG) [1][2]. Returning back to our original query, as the number of inputs increases, the rules capturing the interaction between inputs increases exponentially (rule explosion), making it difficult for humans to debug or back out the operations contributing to a particular crisp output. Further, as Safiotti points out in [3], due to the compounding interactions between these inputs and the inability to use mathematical analysis to profile and evaluate the fuzzy system, it is unlikely for a method besides empirical testing to determine whether the system outputs the desired values.

While these are clear limitations that make fuzzy systems difficult to implement, fuzzy logic is well suited to the domain of robotics, specifically to autonomous robots that can be modeled as nonlinear systems. Uncertainty contributing to this nonlinearity manifests itself in the form of sensor noise (measurement noise), motion noise (process noise), and environmental disturbances. The qualitative property of fuzzy logic makes it uniquely suited to these types of problems, where we can improve decision-making in the presence of uncertainty by combining data from several sources in the rule application stage, like the way that a Kalman filter fuses data from multiple sensors for self-localization [3]. Using fuzzy logic we can build a behavior-based robotic controller: first, we transform sensor values into antecedent linguistic variables like proximity to an obstacle or goal (which may have a range of values like very close, close, far, and very far), then apply IF-THEN rules to map the antecedents to one or more consequent behaviors (follow-line, avoid-obstacles, go-to-goal, etc.), which are then defuzzified to output crisp actuator commands executed by the robot [1]. To avoid the high computational cost associated with rule explosion, fuzzy logic control systems typically split the decision-making process into several hierarchical layers, where each layer is a fuzzy logic controller (FLC) responsible for implementing the control logic for a particular behavior [4]. In this control architecture known as subsumption architecture, layers operate both concurrently and independently. A high-level coordination layer (another FLC) arbitrates between the outputs of each FLC layer, either by suppressing (subsuming the commands of higher-up layers), inhibiting (preventing a layer’s output entirely), or weighting the output commands of each layer to converge on a single actuator command [4][5]. Analogous to how the Kalman filter performs “sensor fusion,” fuzzy logic control systems perform “command fusion” through subsumption architecture.

Though fuzzy logic systems can be an asset for problems with pervasive uncertainty like robotic localization, navigation, and control, it is still a field in its early stages of development. Before the benefits of fuzzy logic can be readily applied to any relevant real-world problem, its limitations of transparency, rule explosion, and requisite domain expertise must be addressed in order to be a viable alternative to classical methods. This is what researchers like Professor Mendel are hard at work for:  developing the formal tools needed to analyze, implement, and advance the emerging field of fuzzy logic systems.
